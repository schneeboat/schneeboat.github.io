Background#

You have been managing a product long enough that you know every detail about the technical stuff, the people that take care of each part, their personalities, etc. Now you need to transfer all the responsibilities to someone else who doesn’t know anything. How to do it methodically?

  • Phase: Operation

  • Scale: Small

  • Complexity: Very complex

No online modern tools for managing the product (*sigh*)

Step 1: Gather documents. Notable examples:#

Dev:#

  • Business Objectives

  • Plannings

  • Requirements Specifications

  • Technical Architecture (if any)

  • Change Management Specifications (if any)

  • UX, UI

  • Source codes (if any)

  • Dev documentation (if any)

  • Bugs, Tasks

  • QA reports

GROUP BY every month (depends)

Ops:#

  • Go-live brochures

  • User Manual

  • Admin Manual

  • FMEA (Failure Modes and Effects Analysis)!!!

  • User Feedbacks

Step 2: Send the above in batches as “Daily Reads”#

Meanwhile, produce the following:

  • A file that contains the name of every team member, and their related responsibilities in the product

  • Contact each party to add permissions for the new person

  • A file that contains EVERY link to all related platforms (test, prod, middlewares, console, user management, etc)

  • A step by step guide to producing product analysis

  • Or write a python script if you have too much time

Step 3: Schedule meetings#

Don’t over-complicate things. Get all middlewares and anything useful ready, then use the product as a normal user, show what happens behind the scenes, explain what could go wrong at each step, why is that, and how to solve it. It must cover every scenario. Record videos.

Step 4: Wait and see#

Be patient and answer all the questions :)
